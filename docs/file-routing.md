# File Routing

`prettier-plugin-salesforce` routes Salesforce-owned source by file path.
Salesforce-only routing is the default and only behavior. There is no option or
CLI flag to widen the router to general project files.

Corpus contracts support these environment variables:

- `SF_PRETTIER_CORPUS_ROOT`: primary corpus root for primary-corpus
  availability and focused primary corpus tests.
- `SF_PRETTIER_DEFAULT_CORPUS_ROOT`: default root for multi-corpus contracts.
- `SF_PRETTIER_EXTRA_CORPUS_ROOTS`: comma/newline list of additional corpus roots.
- `SF_PRETTIER_PROJECT_PACK_ROOTS`: comma/newline list of roots that should
  always expand as "one child directory equals one project root."
- `SF_PRETTIER_PROJECT_PACK_ROOT`: single project-pack root override.
- `SF_PRETTIER_MAX_CORPUS_FILES`, `SF_PRETTIER_CORPUS_SHARD_COUNT`, and
  `SF_PRETTIER_CORPUS_SHARD_INDEX`: local corpus throttles for large runs.

## Support Boundaries

- Routing checks use the filepath extension and path text exactly as provided.
- LWC HTML routing is path-sensitive (`/lwc/` plus `.html`).
- `.html` outside `/lwc/` does not route through the LWC file-route branch.
- Markup parser transforms stay enabled for non-`.html` markup extensions:
  `.page`, `.component`, all `.{cmp,intf,tokens,evt,design,auradoc}` paths, and Aura-path `.app` files.
- Parser transforms for `.html` apply only when either:
  - no filepath is provided, or
  - the filepath matches `.../lwc/<component>/<component>.html`.
- For component-shaped LWC paths, parser dialect inference upgrades `unknown` to `lwc` even when `<template>` tags are weak or missing.
- For non-LWC `.html` paths, the router delegates to Prettier core HTML formatting.
- Metadata XML uses conservative formatting rules; when guard checks fail, the
  formatter returns original XML text instead of reshaping it.
- Metadata family routing is centralized in `src/metadata-families.ts` and shared by
  corpus and benchmark route checks through `routeFile(...)`.
- Custom Labels sorting is opt-in with `salesforceSortLabelsByFullName`.
  Default is `false`, so label order stays as written. The old
  `salesforceSortLabelEntriesByFullName` name remains as a legacy alias.
- Generic XML passthrough stays off-transform unless one of the metadata
  transform-on path triggers matches.
- LWC `.js`, `.ts`, and `.css` remain delegated to Prettier core.
- Aura bundle `.js` and `.css` files remain delegated to Prettier core.
- Staticresource `.js`, `.jsx`, `.ts`, `.tsx`, `.css`, `.scss`, `.html`, `.json`, `.yml`, `.yaml`, and `.md` files remain delegated to Prettier core.
- `StaticResourceSources` `.js`, `.jsx`, `.ts`, `.tsx`, `.css`, `.scss`, `.html`, `.json`, `.yml`, `.yaml`, and `.md` files remain delegated to Prettier core.
- Aura and LWC `.svg` files stay pass-through except for trailing newline normalization.
- Staticresource `.svg` files stay pass-through except for trailing newline normalization.
- Staticresource and `StaticResourceSources` `.py`, `.php`, and `.robot` files stay unknown; this plugin does not own those languages.
- `customMetadata/*.md` routes to metadata XML.
- Salesforce payload text files (`.email`, `.resource`, `.asset`, Agentforce authoring payloads in AI bundle paths, and Salesforce-owned no-extension text basenames such as `.forceignore`) stay pass-through except for trailing newline normalization.
- Ordinary project text such as `.txt`, `.cfg`, `.sql`, `.soql`, `.csv`, `.log`, `.notes`, `.dwl`, `.schema`, shell scripts, `.env`, `.toml`, lockfiles, CODEOWNERS, and pre-commit hooks stays outside the plugin route unless Prettier core owns it.
- Prettier file-info inference maps Salesforce-owned payload basenames (for example `.forceignore`) to the plugin `salesforce-router` parser.
- Encrypted key assets such as `assets/server.key.enc` stay `unknown` by contract, with parser inference held at `null`.
- Unknown routes in corpus contracts cover non-Salesforce paths, binary asset extensions, and encrypted key exceptions.

## `.app` Routing

`.app` is shared by Aura and metadata applications. The plugin now infers
`salesforce-router-by-path` for `.app` and delegates by `routeFile(filepath)`:

- `applications/**/*.app` routes as metadata XML formatting.
- `aura/**/*.app` routes as Aura markup formatting.

## Labels Sort Option

Use this option to sort `<labels>` blocks in `CustomLabels.labels` by nested
`<fullName>` value.

Default:

```json
{
  "salesforceSortLabelsByFullName": false
}
```

Enabled:

```json
{
  "plugins": ["prettier-plugin-salesforce"],
  "salesforceSortLabelsByFullName": true
}
```

## Ambiguous Extensions

Some Salesforce extensions are shared across Salesforce file families. Keep
path-based parser overrides in user config when a path needs deterministic
parser selection.

Prettier infers parsers from language `extensions` and `filenames`, not from
full glob ownership. The plugin therefore registers a small set of shared
extensions (`.html`, `.md`, `.xml`, `.svg`, `.app`) as path-aware router gates.
The router transforms only Salesforce-owned paths and leaves other paths to
Prettier core or `unknown`.

```json
{
  "overrides": [
    {
      "files": "force-app/main/default/applications/**/*.app",
      "options": { "parser": "salesforce-metadata-xml" }
    },
    {
      "files": "force-app/main/default/aura/**/*.app",
      "options": { "parser": "salesforce-markup" }
    },
    {
      "files": "force-app/main/default/customMetadata/**/*.md",
      "options": { "parser": "salesforce-metadata-xml" }
    },
    {
      "files": "*.md",
      "excludeFiles": "force-app/main/default/customMetadata/**/*.md",
      "options": { "parser": "markdown" }
    }
  ]
}
```
